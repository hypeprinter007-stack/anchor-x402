First time? Which sign-in option to use

Sign in with passkey (recommended for almost everyone):

  1. Tap the green Sign in with passkey button.
  2. Your browser/device prompts for Face ID / Touch ID / device PIN.
  3. That creates a self-custodial Coinbase Smart Wallet bound to your device. No app to install.
  4. Fund it with USDC on Base (the trust footer below has links).

Use my browser wallet (desktop power users):

  1. If you have MetaMask / Rabby / Frame / etc. as a browser extension, the button appears below "Sign in with passkey".
  2. One click — your extension prompts; you approve.

We dropped WalletConnect mobile flow. It's fundamentally unreliable for x402 payments — mobile wallet apps lose their relay subscription when backgrounded, and the WC spec provides no soft-recovery path. Coinbase's own x402 reference architecture uses embedded / smart wallets, not WC.

Operator

anchor-x402 — built by Christopher Ferjo. Trust portal · GitHub · Source for this page · ERC-8004 #47261 · OpenAPI

Where your payments go

Treasury: 0x12746…cD0B6 — verified on Basescan. Every payment is a single EIP-3009 USDC authorization you sign per tool call. No subscription, no held balance.

What we can and cannot do

Can: sign USDC payments to the treasury above, scoped to a session cap you control. Cannot: touch other tokens, exceed your cap, custody keys, or sign arbitrary transactions.

Need USDC?

You can fund a wallet with USDC on Base via Coinbase Onramp, MoonPay, or any exchange that supports Base. In-app top-up coming soon.

Anti-abuse

Free chat is rate-limited and bounded by message length so a single user can't burn through our LLM budget. Tool calls require explicit per-call approval — there is no autonomous spend.
